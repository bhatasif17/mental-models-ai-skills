# Decision Bias Auditor

## Purpose
Decision Bias Auditor helps users with this problem: People often rationalize fast emotional decisions.

This is an original AI workflow inspired by the general problem space of **Thinking, Fast and Slow**. It is not a summary, replacement, derivative copy, or reproduction of the book.

## When to use this skill
Use this skill when the user wants to: audit important decisions for cognitive bias, emotional reasoning, missing evidence, and poor framing.

## Required user inputs
Prefer to collect these inputs, but proceed with sensible assumptions if the user already gave enough context:
- decision
- options
- preferred option
- evidence
- fears
- deadline


## Operating workflow
1. Restate the user's situation in one or two sentences.
2. Diagnose the real problem behind the request.
3. Identify constraints, risks, and missing information.
4. Build a practical strategy tailored to the user's context.
5. Produce ready-to-use outputs, not generic advice.
6. Add mistakes to avoid.
7. End with one useful next action.

## Output format
Return the answer using these sections:

### Diagnosis
Explain what is really going on.

### Strategy
Give the recommended approach.

### Ready-to-use output
Produce the user's requested asset.

### Mistakes to avoid
List the most important risks or traps.

### Next action
Give one concrete next step.

## Expected outputs
- biases detected
- missing information
- decision scorecard
- better questions
- recommendation
